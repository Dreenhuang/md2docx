# Markdown批量转DOCX工具 V1.0 产品需求规格文档

## 1. 文档信息

| 项目 | 内容 |
|------|------|
| 文档版本 | V1.0 |
| 创建日期 | 2026-05-25 |
| 技术栈 | Python + PySide6 |
| 目标平台 | Windows 10/11 |
| 分发方式 | 安装程序（.exe安装包） |

## 2. 产品定位

一款Windows桌面端轻量化可视化工具，主打**一键批量Markdown转标准化合规DOCX**，支持用户可视化调整排版格式、自定义输入输出路径，内置固定合规排版模板，自动生成页眉页脚、统一字体版式，实现零人工干预标准化输出。

## 3. 核心功能需求

### 3.1 GUI界面功能

#### 3.1.1 路径配置区
- **输入路径配置**：支持选择单个MD文件、选择整个文件夹（批量读取所有.md文件）
- 显示当前选中路径，支持路径手动输入、路径清空按钮
- **输出路径配置**：支持自定义选择输出文件夹，默认生成独立output文件夹
- 支持"输出至原文件同目录"切换开关
- 实时显示当前输出保存路径

#### 3.1.2 格式参数配置区
- 默认加载官方标准合规模板
- 支持用户手动修改所有参数
- 支持「恢复默认设置」一键重置
- 支持保存自定义配置为模板
- 可配置项：纸张规格、页边距、字体、字号、行距、首行缩进、标题样式、重点内容样式、页眉页脚开关、页码规则

#### 3.1.3 文件列表区
- 导入文件后，列表展示所有待转换MD文件
- 包含：文件名、文件路径、转换状态（待转换/转换中/成功/失败）
- 支持单选、多选、单独删除单文件、清空列表

#### 3.1.4 操作按钮区
- 添加文件、添加文件夹、清空列表
- 开始批量转换、停止转换
- 恢复默认格式、打开输出文件夹

#### 3.1.5 日志与进度区
- 实时显示转换进度（进度条+百分比）
- 实时日志：导入文件数量、转换成功数、失败数、失败原因
- 支持日志清空、日志导出（TXT格式）
- 日志字段：时间戳、文件名、状态、错误详情
- 日志持久化存储到软件目录logs文件夹

### 3.2 单文件转换功能
- 支持选择单个.md文件，一键转换为合规DOCX文件
- 转换完成后自动套用所有版式规则、页眉页脚、重点标注格式

### 3.3 批量转换功能（核心功能）
- 支持批量导入文件夹内所有MD文件，自动遍历读取，过滤非.md格式文件
- 批量串行转换，单文件转换失败不中断整体任务，自动跳过并记录失败日志
- 批量转换完成后统一统计成功率、失败清单
- 所有批量输出文件统一沿用同一套格式配置，版式完全统一
- **单次批量支持20个以内文件**

### 3.4 配置保存与重置功能
- 用户修改的格式参数可保存到本地配置文件（JSON格式）
- 配置存储位置：软件目录
- 提供「恢复默认」按钮，一键还原文档规定的官方标准合规格式
- 支持自定义保存预设模板

### 3.5 输出文件命名规则
- 默认规则：`原文件名_转换日期.docx`
- 同名文件处理：自动重命名（如加序号）
- 保留原中文文件名

## 4. 核心默认转换格式规范

### 4.1 基础页面版式规则
- 纸张规格：A4纸张、纵向排版
- 页边距：左20mm、右20mm、上16mm、下16mm
- 全局颜色：文档所有文字统一**纯黑色**
- 全局字体：全文统一微软雅黑
- 行距规则：全文固定值16磅
- 正文缩进：所有正文段落首行缩进2字符
- 版块间距：标题之间、不同内容版块之间自动保留标准空行

### 4.2 文字样式重点标注规则

#### 4.2.1 标题通用规则
- 所有层级标题（H1-H6）：纯黑色、粗体、字体较Word标准默认字体**加大一号**
- 标题层级编号自动规整，结构统一清晰

#### 4.2.2 重点内容规则
- **通过特定Markdown语法识别重点内容**
- 识别并标准化渲染：关键结论、核心定义、重要数据、注意事项、警示说明、核心总结
- 样式要求：纯黑色、粗体、较正文字体加大一号

#### 4.2.3 结构化元素规则
- Markdown原生项目符号、有序列表、表格、图片、代码块等结构化元素正常保留
- 所有图表、表格内部文字统一纯黑色
- 版式自适应A4页面，无溢出、无变形
- **取消支持流程图**

### 4.3 页眉规范（强制自动生成）
- 页眉内容：当前文档文件标题
- 标题提取优先级：优先Markdown文件中的YAML Front Matter（如 `title: XXX`），无则取第一个一级标题，再无则取文件名
- 对齐方式：居中对齐
- 字体规格：微软雅黑9pt、纯黑色、常规字体
- 生效范围：全文所有页面统一显示

### 4.4 页脚规范（强制自动生成）
- 页脚内容：固定格式「第 X 页 / 共 Y 页 ｜ 编写日期：YYYY年MM月DD日」
- 页码规则：**页码从正文第一页开始编号**，剔除封面/空白页无效页码
- 对齐方式：居中对齐
- 字体规格：小五号微软雅黑、纯黑色
- 日期规则：自动读取当前系统日期，实时生成

### 4.5 字体层级总规范
- 标题类：微软雅黑、加粗、加大一号、纯黑
- 正文类：微软雅黑、常规体、标准字号、纯黑、首行缩进2字符
- 重点内容：微软雅黑、加粗、加大一号、纯黑
- 页眉页脚：统一9pt/小五号微软雅黑、纯黑、居中对齐

## 5. UI界面详细设计规范

### 5.1 整体UI风格
- 轻量化简约办公风，浅色系界面
- 按钮分区清晰，功能分区明确，无冗余元素
- 适配Windows系统原生窗口风格，兼容高清屏幕缩放

### 5.2 界面固定分区（从上至下）
1. **路径配置区**：输入路径选择、输出路径选择、路径快捷按钮
2. **格式配置区**：折叠/展开面板，可调整纸张、边距、字体、行距、页眉页脚开关、重点样式
3. **文件列表区**：展示所有待转换文件及状态
4. **进度日志区**：进度条+实时日志输出
5. **底部操作区**：核心功能按钮

### 5.3 交互细节规范
- 参数修改实时生效，无需重启软件
- 转换过程中禁止关闭窗口、禁止重复点击转换按钮，按钮置灰锁定
- 转换失败文件单独标记红色，成功标记绿色
- 支持转换完成后一键打开输出文件夹
- 一键恢复默认配置有二次确认弹窗，防止误操作

## 6. 非功能需求

### 6.1 性能需求
- 单文件转换耗时≤3秒（常规1万字内MD文档）
- 批量支持单次20个以内文件稳定转换，无卡顿、无崩溃
- 软件启动时间≤2秒

### 6.2 兼容性需求
- 兼容Windows10/Windows11全版本
- 输出DOCX文件兼容Office2016及以上、WPS全版本，打开无格式错乱

### 6.3 稳定性需求
- 异常文件（空文件、损坏MD、乱码文件）自动识别、跳过并记录日志，不崩溃
- 路径非法、权限不足、磁盘空间不足等场景友好弹窗提示
- 批量转换中断后可随时停止任务，资源自动释放

### 6.4 安全性需求
- 纯本地离线运行，无网络请求、无文件上传、无数据采集

## 7. 异常处理机制

- **路径异常**：路径不存在、权限不足、只读目录，弹窗提示「路径无效，请重新选择可读写目录」
- **文件异常**：MD文件损坏、空文件、编码错误，自动跳过，日志记录「文件解析失败」
- **磁盘异常**：磁盘空间不足，终止转换，弹窗提示「磁盘空间不足，无法输出文件」
- **转换中断**：用户手动停止转换，立即终止进程，释放资源，统计已完成文件数量

## 8. 验收标准

1. 完整GUI界面所有功能可正常操作，无按钮失效、无界面错乱
2. 单文件、批量文件转换功能正常，可正常生成.docx文件
3. 所有默认格式规范100%落地，版式、字体、行距、缩进、页边距完全符合要求
4. 页眉、页脚、页码、日期自动生成，规则完全匹配需求
5. 重点内容、标题样式加粗、加大、纯黑规则生效
6. 所有异常场景可友好提示，软件无崩溃、无卡死
7. 输出DOCX文件可正常用Office/WPS打开，无格式丢失、无乱码、无变形
